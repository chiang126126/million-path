# MP500 · AI 策略层 × 确定性风控层 —— 双层架构设计

> 本设计借鉴论文 **WebCryptoAgent: Agentic Crypto Trading with Web Informatics**（arXiv:2601.04687, 2026-01），
> 并按 **500U 起步 + 纯静态/低成本部署 + 风控优先** 的现实约束做了裁剪。
> 配套主文档：`MP500-v1.1.md`。

---

## 0. 先记住论文的实验结论（现实校准）

论文在 BTC/ETH/POL、$10k、一整年（2025-01 → 2026-01）回测多个 LLM：
- **多数模型亏钱**（Gemini 系最差 -48%），最好的夏普约 1.19，胜率 29%–66% 高度分散；
- "加记忆"对有的模型有用、对有的反而更差。

**所以本架构的目标不是"让 AI 帮你赚钱"，而是：**
1. **减少瞎交易**（频繁、低质量信号）；
2. **控制尾部风险**（暴跌/插针/流动性冲击的秒级防护）；
3. **在不确定时主动空仓**（FLAT 也是决策）。

这与 MP500 第一性原则一致：**任何收益目标服从风控；AI 不直接决定下单。**

## 1. 双层解耦架构（核心）

```
            ┌──────────────────────────────────────────────┐
            │   战略层 Strategic Tier（小时级 / 每 8h 刷新）   │
            │   慢、贵、会幻觉 → 只做"参谋"，不直接下单         │
            │                                                │
   新闻 ───►│  News Agent  ┐                                 │
   社交 ───►│  Social Agent ├─► Evidence Document ─► LLM 推理  │─► 决策元组
  OHLCV ───►│  Market Agent ┘   (结构化证据)      (+经验回放)   │  {bias, conf, move, why}
  市场结构─►│                                                │
            └───────────────┬────────────────────────────────┘
                            │  建议（非命令）
                            ▼
            ┌──────────────────────────────────────────────┐
            │   战术层 Tactical Tier · Shock Guard（秒级）     │
            │   快、确定性、无幻觉 → 拥有【否决权】            │
            │   监控：闪崩、插针、点差骤扩、深度枯竭、资金费率异常 │
            │   触发 → 立即止损/撤单/降杠杆/kill switch        │
            └───────────────┬────────────────────────────────┘
                            ▼
                    执行层（先 paper，后 mini live）
```

**铁律:战术层永远可以否决战略层。** LLM 说"加仓",但 Shock Guard 探测到流动性骤降/插针 → 直接拒绝。慢层是参谋,快层是保险丝。

### 在 MP500 阶段里的落点
- **S0–S1**:战略层=你本人 + 看板的 Evidence 快照(见 §3);战术层=预挂止损单 + 人工纪律。**全程 paper / 现货,不用真合约。**
- **S2+**:战术层升级为 private repo 里的**常驻进程**(websocket tick 流 + 确定性规则),才有真正的"秒级"。**注意:秒级 Shock Guard 需要常驻后端,不可能放在 GitHub Pages 静态站。**

## 2. 标准化数据结构

### 2.1 市场快照 Dₜ（每个决策时点构造）
```
D = {
  O: OHLCV（15m + 1h 多尺度）,
  I: 指标 [EMA21, EMA50, EMA200, RSI14, MACD, ATR14, 布林带, VWAP, 昨日高PDH, 昨日低PDL],
  N: 新闻/社交/叙事（含关键人物发言、板块催化剂、跨市场外溢）,
  R: regime 快照（宏观情绪 / 波动状态 / 流动性深度）
}
```

### 2.2 决策元组（战略层唯一合法输出，可审计、可入账）
```
A = { bias, confidence, expected_move_bps, rationale }
  bias            ∈ {LONG, FLAT}   （MP500 保守起见，S2 前不做 SHORT）
  confidence      ∈ [0,1]
  expected_move   预期幅度（基点 bps）
  rationale       为什么——识别到的形态/催化剂
```

### 2.3 regime 滞后阈值（防来回打脸）
- 进场阈值 θ_adopt **>** 出场阈值 θ_hold（**进比出难**，强制持续性）。
- `confidence × P(long) ≥ θ_adopt(R)` 才转 LONG；低于 θ_hold 才转 FLAT；之间维持上一状态。
- **每 8 小时刷新一次 bias**，兼顾适应新行情与稳定性。

## 3. Evidence Document：把论文思路接进我们的静态看板（零成本）

论文要"把多源信息汇成一份结构化证据喂给 LLM"。我们看板**已经有这些数据**（行情/30日线 regime/资金费率/OI/恐惧贪婪/加密新闻/美股宏观/MSTR）。

**做法（已在看板实现，见 `index.html` 的「AI 策略快照」板块）**：
一键把当前所有数据汇成一段**结构化文本**，你复制后粘进任意 LLM（如 Claude），让它按 §2.2 的格式回你一个**决策元组**。

> 这样就实现了论文的"Evidence → LLM 推理"，但：① 零 API 成本、② 无需后端、③ **人 + 风控引擎始终最终拍板**。
> LLM 给的是"参谋意见"，要不要做、做多大，仍由 MP500 风控规则和你决定。

## 4. 反思 / 经验回放（CER）= 你的周复盘升级版

论文每笔交易后在 4h/8h/24h/7d 多周期看结果，让 LLM 产出结构化反思，蒸馏成"经验"存入回放库（30 天半衰期衰减），下次检索相似历史辅助决策。

**MP500 落地（人工版，见 `docs/trade_journal_TEMPLATE.md`）**：
- 每笔交易记录**决策元组**（入场时）+ **结构化反思**（出场后）：
  ```
  reflection = { outcome ∈ {WIN,LOSS,BE}, attribution（技术/新闻/regime 哪个起作用）,
                 lesson（一句话教训）, pattern_validity（这个形态还成立吗）}
  ```
- 攒成一个"经验库"。下次遇到相似 regime，先翻历史同类交易的胜率与教训。
- **这正是 MP500 周复盘的强化版**：从"记账"升级为"可检索的经验"。

## 5. 风控（论文方法 × MP500 红线）

| 论文方法 | MP500 落地 |
|---|---|
| **ATR 波动定仓**：止损距离随 regime 自适应（RISK-ON 仓大止损紧 / RISK-OFF 仓小止损宽） | 用 ATR14 替代固定 2% 止损距离，喂进 §8.1 仓位公式 |
| **分数 Kelly**：置信度 × 统计边缘定仓位，保守缩放上限 | **强约束**：用 1/4 Kelly 甚至更低，且永远 ≤ 单笔风险上限（现货 1%/合约 0.5%）。500U 阶段宁可更小 |
| **熔断**：亏损/回撤阈值停手 | 直接用 MP500 §10 红线（日 2%/周 7%/月 12%/总回撤 20%） |
| **敞口上限**：单标的 & 总权益占比 | 核心池集中，单山寨 ≤5% |
| **时间止损**：流动性恶化/超持有时长自动平 | 战术层规则 |
| **成本闸门**：预期收益 > 手续费+冲击+点差+gas+MEV 才下单 | MP500 §6.2 已有，保留并强制 |

## 6. 信息面的前瞻价值（为什么 News/Social Agent 值得做）

论文的"web informatics"论点，被现实反复印证：
- **关键人物一句话 → 板块分钟级重定价**：如黄仁勋台北发言带飞 MRVL / AI 算力（光互联、芯片、数据中心）相关标的；
- **美股 AI 叙事外溢到加密 AI 代币**：跨市场资金联动越来越快，像极了币圈"大佬喊单"文化。

**落地启示**：
1. Evidence Document 必须含 **新闻 + 叙事 + 关键人物发言 + 跨市场催化剂** 一路（我们看板的加密新闻 + 美股板块 + events.json 已覆盖大部分）；
2. 把 **AI 算力链（NVDA / MRVL / AVGO 等）当作加密 AI 叙事的领先指标**之一，纳入观察；
3. **但仍服从风控**：叙事driven 行情来得快去得也快，只在 regime + 成本闸门 + 风控同时达标时才参与，且优先用卫星仓小仓位。

## 7. 分阶段采纳路线（不要一步到位）

| 阶段 | 战略层 | 战术层 | 反思 |
|---|---|---|---|
| **现在 (S0/S1)** | 你 + 看板 Evidence 快照 → 手动让 LLM 出决策元组（paper 记录） | 预挂止损 + 人工纪律 | 周复盘用结构化反思模板 |
| **S2** | 固化 Evidence 模板 + 8h 刷新节奏 + 滞后阈值 | private repo 常驻进程：websocket tick + 确定性 Shock Guard | 经验库可检索 |
| **S3+** | 评估是否接入 LLM API 自动出建议（仍不自动下单） | Shock Guard 加自动撤单/降杠杆 | CER 半自动 |

> **绝不要**在验证期就让 LLM 自动下单——论文里多数模型是亏的。先 paper、先攒正期望、先让快层有否决权。
