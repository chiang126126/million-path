# MP500 / Million Path — 策略与代码审计报告

审计日期：2026-06-18 · 审计对象：MP500 策略文档（原始 docx）+ v1.1 修订 + 本项目代码

> 结论先行：**策略框架与核心公式正确、风控完整、方向稳健**。原文档最大的优点是「承认目标不能被保证」。审计中发现 4 处需要修正/补强，已在 v1.1 落实。

---

## A. 通过项（无误）

| # | 审计点 | 结论 |
|---|---|---|
| A1 | 仓位公式 `risk_usdt = equity*risk_pct` → `notional = risk_usdt/stop_dist` → `margin = notional/lev` | ✅ 正确。验证：止损触发亏损 = notional×stop_dist = risk_usdt（恒等成立） |
| A2 | 「杠杆不降低风险，只降低保证金」 | ✅ 正确。lev 仅改变 margin，止损金额恒为 risk_usdt |
| A3 | 风控优先级（收益服从风控、不爆仓/不全仓/不马丁/不扛单） | ✅ 正确且关键 |
| A4 | 核心池限定 BTC/ETH/SOL，山寨设为默认关闭的卫星仓 | ✅ 正确，利于自动化与控滑点 |
| A5 | 合约成本门槛（手续费+资金费+滑点+最小盈亏比） | ✅ 正确，必须保留 |
| A6 | 逐仓 only、禁用全仓；止损先于强平 | ✅ 正确 |
| A7 | 熔断阶梯（日/周/月/总回撤 + 连亏计数） | ✅ 完整可执行 |

**公式复算（500U 示例）**：
```
risk_usdt = 500 × 0.005 = 2.5 U
notional  = 2.5 / 0.02  = 125 U
margin    = 125 / 2     = 62.5 U
止损亏损   = 125 × 0.02  = 2.5 U == risk_usdt  ✓
```

## B. 需修正/补强项（已在 v1.1 落实）

| # | 问题 | 风险 | v1.1 修正 |
|---|---|---|---|
| B1 | 目标表述把「60–80 天破百万」当节奏 | 时间压力会逼迫违反风控 | 改为**远期里程碑（非时间承诺）**，长期锚定周化 2–3% |

| B4 | 缺现实收益预期 | 无法量化复盘、易高估 | 新增 §4 收益预期表（按行情分档）+ §4.1 资金成长参考区间 |

## D. 仍需人来把关的风险（代码无法替你承担）

1. **市场风险**：黑天鹅、交易所暴雷、稳定币脱锚——再好的风控也无法 100% 规避。
2. **执行纪律**：系统能拦住程序化违规，但「手动改参数/情绪化干预」要靠每周复盘的 violations=0 来约束。
3. **过拟合**：paper 表现好 ≠ 实盘好。S0→S1→S2 的逐级放量就是为了对抗这一点。
4. **收益不可承诺**：本审计确认的是「策略与代码无误」，**不是**「一定能到 100 万」。
